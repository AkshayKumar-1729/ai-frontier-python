Snippets 1–2 · The Brain
An LLM is a brain in a jar.
Which brain you wake depends on how you ask.
Naive prompt

+ Persona & constraints
"You are a born-and-raised Kodava guide who hates tourist traps. Exactly 3 stops, with opinions." → alive, specific, yours.
Why it's wild
Same model, same question — but a persona + hard constraints changes everything. This is exactly the prompt engineering you did on Day 4 — now you're seeing it in code, driving a real app.

Snippet 7 · The Framework — where you go next
You ran the loop by hand.
The industry declares it — and lets the framework run it.
By hand (Snippet 5)
You wrote the Think → Act → Observe loop yourself: build history, check for a function_call, run it, append the result, repeat.
With Google ADK
You declare an Agent with a model + tools, hand it to a Runner — the framework runs the loop for you.
Why it's wild
Same Wanderlust agent, ~15 lines, no hand-written loop. ADK is Google's open-source Agent Development Kit — the same toolkit used to build production agents. Today it's a teaser; it's the doorway to multi-agent systems, sessions, and deployment.

Snippet 8 · Chain-of-thought
Ask for a fast Yes/No on a budget math problem → it slips. Add "think step by step" → it shows its work and fixes itself. Reasoning is something you can switch on.
Snippet 9 · RAG with embeddings
Remember Day 6 embeddings? Turn private "deals" into vectors, find the closest match to a question, feed only that to the model. This is what embeddings are FOR — giving your AI private memory.
⚡ RAG (Retrieval-Augmented Generation) is how real assistants answer questions about your documents — data the model was never trained on. You just built a tiny one with NumPy and a dot product.
